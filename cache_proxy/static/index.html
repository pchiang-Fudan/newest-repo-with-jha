How Adaptive ML Token-Routing Works

        

          Middle-Out learns which repo context matters, ranks it before each
          model call, and sends fewer tokens to the API without retraining the LLM.
        


        

          

            
1. Repo Signal Ingestion

            

              Reads paths, symbols, tests, workflows, and task intent.
            

          

          

            
2. Self-Learning Context Ranker

            

              Scores files and snippets using repo-specific learned weights.
            

          

          

            
3. Cache-Aware Prompt Geometry

            

              Places stable context where provider caches can reuse it.
            

          

          

            
4. Guardrailed Token Routing

            

              Protects required files while choosing the lowest-cost context plan.
            

          

        


        

          Middle-Out Token-Routing is a modern token-layer analogue of middle-out
          compression: adaptive ML compresses the context stream before it reaches
          the model.
        


        

          
Adaptive ML Repo Memory

          

            Learns from accepted answers, repeated file usage, explicit file
            references, and missing-context corrections. Common paths get cheaper;
            risky tasks get more context.
